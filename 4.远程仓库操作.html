4.远程仓库操作

远程仓库操作

1> 查看远程仓库: git remote

   git remote -v :  你也可以指定选项 -v，会显示需要读写远程仓库使用的 Git 保存的简写与其对应的 URL。 

2> 克隆远程仓库到本地: git clone "url(远程仓库的地址)"

   >  如果你使用 clone 命令克隆了一个仓库，命令会自动将其添加为远程仓库并默认以 “origin” 为简写

3> 添加远程仓库:  git remote add <shortname> <url>  添加一个新的远程 Git 仓库，同时指定一个你可以轻松引用的简写(取个名字)
    

    现在可以使用 eden ---> 这个名字来代替整个 url 了!

4> 从远程仓库中抓取或拉取

    1> git fetch [remote-name] :  这个命令会访问远程仓库，从中拉取所有你还没有的数据。 执行完成后，你将会拥有那个远程仓库

       中所有分支的引用，可以随时合并或查看. 它并不会自动合并或修改你当前的工作
       

    2>  git pull <远程主机名> <远程分支名>:<本地分支名>   --->  可以使用 git pull命令来自动的抓取

        然后合并远程分支到当前分支
           

5> 推送到远程仓库(同步到远程仓库):  git push [remote-name] [branch-name]

   git push edenHao master


6> 查看详细的远程仓库信息: git remote show "远程仓库名称"
